- **コードコメントおよびJavadocの指針**  
  処理の「意図」や設計の「なぜ（Why）」を重視し、AIや開発者が文脈を即座に理解できる平易な表現を用いる。  
  単にコードの挙動をそのままなぞるだけの、不要なノイズコメントは避ける。

- **インデントと書式**  
  インデントは標準的な半角スペース4つとする。

- **型安全性の徹底**  
  ジェネリクスの型引数は省略せず明示し、厳格な型安全性を維持するものとする。  
  可読性向上のための `var` による型推論の使用は、  
  右辺から型が自明な場合（`new` やファクトリメソッド等）に限るものとする。

- **ファイル末尾の改行**  
  すべてのソースコードおよび設定ファイル（`.xml`, `.yml`, `.md`等を含む）の末尾は、  
  POSIX規約に基づき必ず改行コード（LF）で終了させるものとする。
