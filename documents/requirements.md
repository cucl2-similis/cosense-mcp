# 要件定義書：Cosense MCP Server

## 1. 背景と目的

- **背景**  
  日々の技術メモを私用の [Cosense（旧Scrapbox）](https://scrapbox.io/) に蓄積しているが、  
  プライベートプロジェクトであるため外部のAIツールから直接読み込ませることができない。  
  AIとの対話に自身の貴重なナレッジを安全に活用したい。

- **目的**  
  Java / Spring AI を用いて、[MCP（Model Context Protocol）サーバー](https://modelcontextprotocol.io/docs/learn/server-concepts)を自作する。  
  各種AIツールが、安全かつ的確に個人のプライベートなCosenseノートを参照・全文検索できる環境を構築する。

## 2. ユーザーペルソナ & ユースケース

- **ペルソナ**  
  開発メモをCosenseに集約しており、AIツールを日常的に使うエンジニア（自身）。

- **主要なユースケース**
  1. AIに「以前試した、Spring BootのCORSエラーの解決策について書いてあるノートを探して」と指示し、  
     AIが本文まで含めて検索・参照する。
  2. AIに「Cosenseにある〇〇の設計スタンスに基づいて、新しいクラスのコードを書いて」と指示する。

## 3. 機能要件 (スコープ)

将来的な書き込み拡張を考慮しつつ、フェーズ1では参照系機能を確実に実装する。対象プロジェクトは1つに固定する。

### 【Must】フェーズ1：参照機能（初期リリース）

- **キーワード全文検索 (`cosense_search_pages`)**  
  固定されたプロジェクト内を検索し、タイトルだけでなく「本文」も含めてマッチしたページの一覧を返す。

- **ページ内容の取得 (`cosense_get_page`)**  
  指定されたタイトルのページ本文を、文字数制限なしで丸ごとすべて取得して返す。

### 【Want】フェーズ2：書き込み機能（将来的な拡張）

- **新規ページ作成 / 追記**  
  AIとの会話ログや要約を、AI自身が自動でCosenseにページ作成・追記できる基盤。

## 4. 非機能要件

- **動作環境**  
  本サーバーはローカルマシン上で動作させることを前提とし、外部ネットワークへの公開は行わない。  
  MCP STDIO プロトコルを使用し、AIクライアントツールのサブプロセスとして起動される。

- **セキュリティ**  
  プライベートプロジェクトへのアクセスに必要なブラウザのセッションID（`connect.sid`）は、環境変数から安全に読み込む。  
  認証情報をソースコードや設定ファイルにハードコードせず、ログ出力・エラーメッセージにも含めない。

- **データ保持**  
  AIが文脈を完全に理解できるよう、テキストの省略や切り詰めは行わず全量を渡す。

- **サーバー負荷への配慮と安定性**  
  短時間に大量のリクエストを送信してCosenseのサーバーに負荷をかけないよう、適切なウェイト制御を行う。  
  また、4xxや5xxエラー発生時には適切なエラーハンドリングを行う。
